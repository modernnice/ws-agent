# Get Deck Recipe Skill

## 概述 (Overview)
本技能提供对 Weiss Schwarz 官方网站卡组列表 (Deck Recipe) 的检索与解析能力。
核心功能是通过逐级访问官网页面 (`https://ws-tcg.com/deckrecipe/`)，帮助用户查找特定的卡组信息，并支持查看卡组内的具体单卡详情。

## 约束与规则 (Constraints)
- **必须优先使用工具**：执行查询时，**必须且只能**调用 `fetch_web_content` 工具来获取网页内容。
- **禁止猜测 URL**：严禁凭空构造未经验证的 URL。必须先访问索引页，从页面中提取真实的子链接后才能进行下一步访问。
- **逐步引导**：由于卡组层级较深（索引页 -> 分类页 -> 卡组列表 -> 卡组详情），**禁止**一次性尝试获取所有层级的内容。必须通过多轮对话引导用户逐步深入。
- **URL 规范化**：官网链接通常是相对路径（如 `/deckrecipe/detail/...`），调用工具前**必须**将其转换为绝对路径（`https://ws-tcg.com` + path）。
- **语言处理**：官网内容主要为日语，向用户汇报时**必须**翻译或概括为中文。

## 执行流程与指令 (Instructions)

### 1. 意图识别 (Intent Recognition)
当用户输入包含以下意图时，**必须**调用此技能：
- **查最新卡组**: "最近有什么新卡组", "看看最新的上位卡组"
- **查特定赛事**: "WGP2023 冠军用的什么卡组", "查看某某比赛的卡表"
- **查系列卡组**: "有没有某某系列的构筑参考"

### 2. 执行步骤 (Execution Steps)

#### Step 1: 初始访问 (Initial Access)
- **动作**: 调用 `fetch_web_content("https://ws-tcg.com/deckrecipe/")`。
- **解析**: 提取页面中的主要卡组分类链接（例如 "WGP2023 世界大会", "某系列发售纪念大会" 等）。
- **反馈**: 向用户展示这些分类，并询问用户想看哪一个。

#### Step 2: 进入分类 (Enter Category)
- **动作**: 根据用户选择的分类 URL，再次调用 `fetch_web_content(url)`。
- **解析**: 解析该分类页面下的具体卡组列表（例如 "冠军 A 的卡组", "亚军 B 的卡组"）。
- **反馈**: 提取卡组的详细页面链接，向用户展示具体的卡组选项。

#### Step 3: 查看卡组 (View Deck)
- **动作**: 根据用户选择的卡组 URL，调用 `fetch_web_content(url)`。
- **解析**: 
  - 解析卡组详情页，列出卡组中的关键卡牌（Climax, Level 3, Level 0 等）。
  - 提取关键卡牌的名称和数量。
- **反馈**: 
  - 展示卡组构成。
  - 如果用户对某张特定卡牌感兴趣，可以结合 `card-search` 技能进行进一步查询。

### 3. 结果处理与反馈 (Response Generation)
工具返回 Markdown 结果后，按以下格式处理：

#### A. 列表展示 (List View)
当展示卡组分类或列表时，使用清晰的列表格式：
- **[分类/卡组名称]**: [简要描述]
- ...

#### B. 卡组详情 (Deck Detail)
当展示具体卡组时，按等级或类型分组展示：
- **Level 3**: [卡名] x[数量], ...
- **Level 1**: [卡名] x[数量], ...
- **Climax**: [卡名] x[数量], ...
- **URL**: [该卡组的原始链接]

#### C. 异常处理
- **访问失败**: "无法访问该页面，可能是网络问题或链接失效。"
- **解析失败**: "无法解析该页面的卡组信息，请尝试其他链接。"
